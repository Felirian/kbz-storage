# Архитектура системы Aurora

## Общая архитектурная концепция

Система Aurora реализована на основе клиент-серверной архитектуры.

Frontend-приложение взаимодействует с backend-сервисом через REST API.

Описание целей системы приведено в документе [Паспорт проекта](/projects/aurora/passport).

## Основные компоненты системы

### Frontend

Frontend отвечает за пользовательский интерфейс и взаимодействие пользователя с системой.

Основные технологии:

- React
- TypeScript
- REST API

Требования к пользовательскому интерфейсу описаны в [Техническом задании](/projects/aurora/technical-specification).

### Backend

Backend-часть системы реализует:

- бизнес-логику приложения;
- обработку пользовательских запросов;
- взаимодействие с базой данных.

### База данных

В качестве основной СУБД используется PostgreSQL.

База данных хранит:

- статьи базы знаний;
- информацию о пользователях;
- задачи проекта.

## API

Взаимодействие клиента и сервера осуществляется через REST API.

Каждый запрос проходит проверку авторизации пользователя. Права пользователей определяются в соответствии с ролями, описанными в статье [Роли участников проекта](/projects/aurora/roles).
