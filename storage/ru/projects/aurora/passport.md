# Паспорт проекта Aurora

## Общая информация

**Название проекта:** Aurora  
**Тип системы:** Веб-платформа управления цифровыми продуктами  
**Цель проекта:** создание единой платформы для управления задачами, документацией и релизами внутри продуктовых команд.

Aurora разрабатывается как внутренняя корпоративная система, предназначенная для повышения прозрачности процессов разработки и упрощения взаимодействия между командами разработки, дизайна и менеджмента.

Подробные требования к системе описаны в документе [Техническое задание](../requirements/aurora-technical-specification.md).

## Заказчик

Внутренний заказчик — департамент цифровых продуктов компании.

Основная задача заказчика — получить инструмент, который позволит:

- централизовать проектную документацию;
- ускорить поиск информации;
- обеспечить единый формат хранения знаний.

Подробное описание требований заказчика приведено в разделе [Цели и задачи проекта](../requirements/aurora-technical-specification.md#цели-и-задачи-проекта).

## Участники проекта

Проект реализуется кросс-функциональной командой.

### Основные роли

**Product Owner**  
Отвечает за формирование продуктового видения и приоритизацию задач. Подробнее о роли можно прочитать в статье [Роли участников проекта](../management/project-roles.md).

**Project Manager**  
Контролирует сроки реализации проекта и координирует работу команды.

**Команда разработки**  
Отвечает за реализацию архитектуры и программной логики системы. Архитектурные решения описаны в статье [Архитектура системы Aurora](../technical/aurora-architecture.md).

**Команда дизайна**  
Формирует пользовательский интерфейс и дизайн-систему продукта.

## Технологический стек

Основные технологии проекта:

- Frontend — React + TypeScript
- Backend — Node.js
- База данных — PostgreSQL
- API — REST

Архитектурная схема и взаимодействие компонентов описаны в документе [Архитектура системы Aurora](../technical/aurora-architecture.md).

## Основные этапы проекта

Проект разделен на несколько этапов:

1. Исследование требований
2. Проектирование архитектуры
3. Разработка MVP
4. Тестирование
5. Запуск системы

Подробный список функциональных требований представлен в документе [Техническое задание](../requirements/aurora-technical-specification.md).
