# API-взаимодействие в Forge

## Транспорт

Forge общается с бэкендом по REST поверх HTTPS. WebSocket в проекте не используется — обновления состояния маршрутов согласования приходят через серверные события (SSE) на отдельном эндпоинте.

| Канал | Назначение |
|---|---|
| REST | основной CRUD по схемам, ответам, маршрутам |
| SSE (`/events/route/:id`) | live-обновление статуса конкретного маршрута согласования |

## Структура клиента

```
src/shared/api/
  http.ts            # axios instance + interceptors
  sse.ts             # обёртка над EventSource
  errors.ts          # классы ошибок
src/modules/<module>/api/
  <resource>.api.ts  # вызовы к бэкенду
  <resource>.swr.ts  # SWR-хуки
```

В клиентских компонентах прямой axios запрещён ESLint-правилом — только через хуки. В RSC выполняется прямой fetch с использованием cookies из `next/headers`.

## Конвенция запросов

REST-эндпоинты придерживаются ресурс-ориентированного стиля.

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/v1/forms` | список форм организации |
| POST | `/api/v1/forms` | создание новой формы (черновик) |
| GET | `/api/v1/forms/:id` | получить форму со схемой |
| PATCH | `/api/v1/forms/:id` | обновить метаданные |
| PUT | `/api/v1/forms/:id/schema` | обновить схему (целиком) |
| POST | `/api/v1/forms/:id/publish` | публикация (создаёт версию) |
| POST | `/api/v1/forms/:id/responses` | отправка заполненной формы |
| GET | `/api/v1/forms/:id/responses` | ответы (пагинация) |
| POST | `/api/v1/routes` | создание маршрута |
| GET | `/api/v1/routes/:id` | состояние маршрута |
| POST | `/api/v1/routes/:id/steps/:step/decide` | решение по шагу (approve/reject) |

Пагинация — cursor-based. Все коллекции возвращают `{ data, nextCursor, total }`.

## Аутентификация

JWT в `httpOnly` cookie. Особенность Forge — двойная среда исполнения (RSC и client):

- На сервере (RSC и API routes Next.js) cookie читается через `cookies()` и проксируется в `Authorization: Bearer ...`.
- На клиенте axios автоматически отправляет cookie (same-origin), и interceptor дёргает рефреш при 401.

Refresh-токен и access-токен — короткоживущие, ротация при каждом обновлении.

## Обработка ошибок

Класс ошибки нормализуется единообразно:

```ts
class ApiError extends Error {
  code: 'VALIDATION' | 'NOT_FOUND' | 'CONFLICT' | 'INTERNAL' | 'NETWORK' | string;
  status: number;
  details?: unknown;
}
```

Поведение в UI:

- **VALIDATION (422)** — Forge показывает ошибки рядом с полями (если ответ содержит карту `field → messages`);
- **CONFLICT (409)** — специальная обработка для редактора: если схема была изменена кем-то другим, пользователю предлагается пересмотреть изменения. Реализовано через ETag (см. ниже);
- **NOT_FOUND (404)** — экран-заглушка ресурса;
- **INTERNAL (5xx)** — глобальный тост, для GET — автоматический ретрай;
- **NETWORK** — баннер «нет соединения», ретрай при восстановлении.

## Optimistic concurrency

Схемы форм могут редактироваться параллельно (например, два администратора в одной организации). Применяется ETag:

1. GET `/forms/:id` возвращает заголовок `ETag: "v17"`.
2. PUT `/forms/:id/schema` отправляет `If-Match: "v17"`.
3. Если бэкенд видит более свежую версию — возвращает 409 Conflict.

UI редактора в этом случае показывает диалог, где пользователь видит свежую версию и решает, как объединить.

## SSE для маршрутов

SSE-соединение поднимается на странице конкретного маршрута. На мобильных и нестабильных сетях SSE может рваться — клиент автоматически переоткрывает с экспоненциальным backoff (1с → 30с).

> ⚠️ Внимание: SSE передаёт обновления **только пока страница открыта**. Для надёжного уведомления о решении по шагу используется push-уведомление на e-mail на уровне бэкенда.

## Idempotency

Мутации, которые могут быть повторены пользователем (публикация формы, отправка маршрута на согласование), принимают `Idempotency-Key`. Клиент генерирует UUID v4 и кэширует его до завершения операции.

## Логирование и мониторинг

Каждый запрос с длительностью > 2 секунд логируется как Sentry breadcrumb. Ошибки 5xx и сетевые — как issue. На стороне Datadog APM настроены алерты на error-rate и latency публичных эндпоинтов.
