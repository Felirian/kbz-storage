# Структура команды Pulse

## Состав

Команда проекта — четыре человека, выделенных полностью на Pulse, плюс привлекаемые на ревью эксперты из других проектов.

| Роль | Сотрудник | Опыт | Зона ответственности |
|---|---|---|---|
| Project Manager | [Сергей](/team#сергей) | 4 года | Сроки, спринты, риски, коммуникация с заказчиком |
| Middle Frontend | [Кирилл](/team#кирилл) | 3 года | Архитектура модулей, код-ревью, интеграция с API, менторинг |
| Junior Frontend | [Алексей](/team#алексей) | 1 год | Вёрстка, компонентная разработка, исправление багов |
| UX/UI Designer | [Олег](/team#олег) | 3 года | Дизайн-система, компонентная библиотека, адаптивность |

Привлекаемые роли: Senior Frontend ([Марина](/team#марина)) — консультации по перформансу один-два раза в спринт; Dev Lead ([Андрей](/team#андрей)) — ревью архитектурных PR; Designer Lead ([Елена](/team#елена)) — аудит визуального соответствия раз в квартал.

## Матрица RACI

R — Responsible (исполнитель), A — Accountable (ответственный), C — Consulted (консультирует), I — Informed (информируется).

| Процесс | PM | Middle | Junior | Designer |
|---|---|---|---|---|
| Планирование спринта | A | R | C | C |
| Разработка фичи | I | A/R | R | C |
| Код-ревью | I | A | R | — |
| Дизайн макетов | I | C | I | A/R |
| Передача макетов в разработку | I | R | C | A |
| Релиз в production | A | R | I | I |
| Коммуникация с заказчиком | A/R | C | I | I |
| Реакция на инциденты | A | R | I | C |

## Зоны ответственности

**[Сергей](/team#сергей) (PM).** Ведёт спринты в Jira, готовит еженедельный статус для заказчика, отвечает за выполнение SLA по реакции на баги (см. [Процессы](/projects/pulse/processes)).

**[Кирилл](/team#кирилл) (Middle).** Технический владелец репозитория, апрувит все PR, поддерживает архитектурный документ актуальным (см. [Архитектура приложения](/projects/pulse/architecture)). Является техническим контактом для бэкенд-команды заказчика.

**[Алексей](/team#алексей) (Junior).** Работает по задачам из спринта под менторством [Кирилла](/team#кирилл). Отвечает за компоненты UI и баг-фиксы. На ревью присутствует как читатель: смотрит чужие PR без права апрува.

**[Олег](/team#олег) (UX/UI).** Поддерживает дизайн-систему Pulse в Figma и Storybook (см. [UI-библиотека](/projects/pulse/ui-library)). Отвечает за адаптивность всех новых экранов и за чек-лист хендоффа (см. [Регламент передачи макетов](/projects/pulse/handoff-regulation)).

## Каналы коммуникации

- `#pulse-dev` — рабочий канал команды;
- `#pulse-alerts` — алерты от мониторинга и CI;
- `#pulse-client` — закрытый канал для общения с заказчиком, доступ имеют PM и Dev Lead;
- ежедневный стендап в 10:30 (15 минут).

## Эскалация

Технические блокеры → [Кирилл](/team#кирилл) → [Андрей](/team#андрей) (Dev Lead).
Продуктовые вопросы → [Сергей](/team#сергей) → [Наталья](/team#наталья) (PO).
Спорные дизайн-решения → [Олег](/team#олег) → [Елена](/team#елена) (Designer Lead).
