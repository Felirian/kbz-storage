# Стайлгайд Bridge

## Назначение

Bridge — портал для разработчиков, поэтому стайлгайд оптимизирован под технический контент: документацию, code samples, табличные данные. Эстетика — спокойная, контрастная, минимум декора.

### Палитра кода

Подсветка синтаксиса использует свою палитру:

| Токен | Назначение |
|---|---|
| `--code-keyword` | ключевые слова |
| `--code-string` | строки |
| `--code-number` | числа |
| `--code-comment` | комментарии |
| `--code-property` | имена свойств |
| `--code-tag` | теги, методы HTTP |

Цвета настроены под две темы (light / dark), переключаются через `data-theme` на `<html>`.

> ⚠️ Внимание: для документации читаемость важнее всего. Базовый размер `body` в Bridge — 15px, на 1px больше, чем в [Pulse](/projects/pulse/styleguide) и [Forge](/projects/forge/styleguide).

## Сетка и отступы

- Контентная область — 12 колонок, gutter 24px.
- Документация — three-column layout (TOC слева, контент по центру, anchors справа), max-width контента — 768px.
- Sandbox и playground — на всю ширину.
- Базовая единица отступов — 4px. Шкала: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 56 / 80.
- Радиусы: `--radius-sm: 4px`, `--radius-md: 6px`, `--radius-lg: 12px`, `--radius-pill: 999px`.

## Иконки

Lucide. Размеры — 14, 16, 20, 24. Дополнительно — иконки методов HTTP (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`) — это специальные badge'и с цветом по семантике.

## Тёмная тема

Тёмная тема — обязательная в Bridge (целевая аудитория — разработчики). Все токены имеют пары для `data-theme="dark"`.

Принципы:

- фон — `--color-neutral-900` (`#0F172A`);
- основной текст — `#E2E8F0`;
- контраст соответствует WCAG AA;
- блоки кода имеют отдельную тему подсветки (One Dark Pro inspired).

## Принципы применения

- В одном экране — один уровень акцента (primary). Дополнительные действия — secondary или ghost.
- Семантические цвета не используются декоративно.
- В коде badge'и методов всегда одного цвета: GET — синий, POST — зелёный, PUT/PATCH — жёлтый, DELETE — красный.
