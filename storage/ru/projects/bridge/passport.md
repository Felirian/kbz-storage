# Паспорт проекта Bridge

## Общая информация

**Название:** Bridge
**Тип системы:** интеграционный портал для внешних API-партнёров
**Статус:** pre-launch (релиз запланирован на конец Q3 2025)
**Цель:** предоставить внешним разработчикам-партнёрам единую точку для работы с публичным API NovaTech: документация, sandbox, управление ключами, мониторинг квот.

Bridge — стратегический проект студии. Он закрывает потребность отдела партнёрств: до Bridge документация была разрозненной, а ключи выдавались вручную через тикеты.

## Заказчик и пользователи

Внутренний заказчик — отдел партнёрств NovaTech. Конечные пользователи — внешние разработчики компаний-партнёров. Отдельная роль — администраторы со стороны партнёра, которые управляют ключами и квотами своей организации.

Ключевые ожидания:

- быстрый старт: разработчик должен сделать первый успешный запрос за < 5 минут;
- актуальность документации, синхронная с релизами API на бэкенде;
- безопасное управление ключами с возможностью ротации без даунтайма.

## Бизнес-метрики

Так как Bridge ещё не в production, метрики формируются из закрытого пилота с пятью партнёрами.

| Метрика | Цель к запуску |
|---|---|
| Time-to-first-call | < 5 мин |
| Доля партнёров, перешедших с тикетов на портал | 80% |
| Doc satisfaction (внутренний опрос) | ≥ 4 / 5 |
| Uptime sandbox | 99.5% |

После релиза целевые метрики будут пересмотрены — см. [Беклог и roadmap](/projects/bridge/backlog-roadmap).

## Технологический контекст

Bridge — Vite + React приложение со встроенной MDX-документацией, Swagger UI для интерактивных спецификаций и Monaco Editor для playground. Авторизация — OAuth 2.0 с PKCE. Подробнее — в [Используемые технологии](/projects/bridge/tech-stack).

## Ключевые даты

- Q1 2025 — closed alpha с двумя партнёрами;
- Q2 2025 — расширенный пилот (пять партнёров), доработка документации;
- Q3 2025 — публичный релиз, индексация документации поисковыми системами;
- Q4 2025 — расширение sandbox-сценариев и интеграция с биллингом партнёров.

## Риски

> ⚠️ Внимание: основной риск — рассинхронизация документации с реальным API. Митигация — генерация спецификаций из бэкенд-репозитория и сборка в CI Bridge. Ручные тексты допустимы только в концептуальных гайдах. Подробности — в [Архитектура приложения](/projects/bridge/architecture).

Второй риск — безопасность портала: ключи партнёров — чувствительный актив. Механизмы защиты описаны в [API-взаимодействие](/projects/bridge/api-integration). Ответственный — [Андрей](/team#андрей).
